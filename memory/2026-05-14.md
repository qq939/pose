# 2026-05-14

- 用户要求修复 Docker 内部部署的 8082 YOLO Pose Web App：手机端访问、检测结果黑屏、Python venv 缺失、`user_start.sh` 启动脚本。
- 设计为：Web 服务先启动，Python venv 后台安装；安装期间 8082 首页显示安装进度，`/api/setup-status` 返回 JSON 状态，安装完成后自动显示正常页面。
- 修复黑屏：前端 canvas 在没有检测结果或检测服务还没返回时继续绘制源视频帧；Python 视频处理进度改写 stderr，stdout 只保留最终 JSON。
- 注意提交时不能包含 `.venv/`、`node_modules/`、`uploads/`、`dataset/`、`logs/`。
- 用户追加要求摄像头 FPS 控件实时控制摄像头输出。实现思路：rAF 采集最新帧进入 5 张缓存池，按 `1 / fps` 秒消费最新帧输出到结果画布并发起检测，旧帧丢弃；默认 1 FPS。
- 用户反馈状态一直显示“检测服务准备中”。排查本地 `/api/status` 已 `pythonReady: true`，假摄像头检测接口返回 200，判断是前端曾经捕获错误后没有在成功检测帧清回“检测中...”。已计划修复成功帧状态恢复。
- 用户反馈“检测服务异常：stream worker exited code 1”。排查发现 stream worker 退出会被 Node 原样返回前端；已改为 Python stream 每帧捕获异常并返回 JSON，Node 在 worker 掉线时自动回退到单帧检测，避免 UI 直接暴露 worker 退出码。
- 用户反馈视频处理 `Processing failed` 信息不够，并要求视频 Pose 检测结果可下载。实现：`logs/run.log` 增加 `[process-video]` debug 日志，成功结果写入 `uploads/results/*.json`，前端显示下载按钮。
